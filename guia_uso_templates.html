Guia de Uso dos Templates de Automação

Olá! Este guia rápido foi criado para ajudar você a começar a usar os templates de projetos de automação de teste que você baixou. O objetivo é facilitar a escrita e a execução de testes automatizados, mesmo que você não seja um programador.

Este guia irá te ajudar a entender a estrutura básica do projeto e como dar os primeiros passos para rodar seus testes.

Como Começar

  1. Baixe e Extraia: Baixe o arquivo ZIP do template e extraia todo o conteúdo para uma pasta no seu computador.
  2. Abra em um Editor: Use um editor de código moderno (como VS Code, Sublime Text ou outro de sua preferência) para abrir a pasta onde você extraiu os arquivos.
  3. Instale as Dependências: Siga as instruções no arquivo README.md para instalar as bibliotecas necessárias (geralmente com um comando como npm install ou pip install).
  4. Configure a URL Base: Para que os testes saibam qual site visitar, você precisa configurar a URL principal. Procure por essa configuração nos seguintes arquivos, dependendo do seu template:
    • Cypress: No arquivo cypress.config.ts, encontre a opção baseUrl e troque o endereço de exemplo pela URL do seu site.
    • Playwright: No arquivo playwright.config.ts, dentro da seção use, encontre a opção baseURL e troque o endereço de exemplo pela URL do seu site.
    • Robot Framework: No arquivo resources/variables.robot, edite a linha que define ${BASE_URL} para a URL do seu site.
  5. Explore a Estrutura: Entenda onde estão os arquivos de teste e os arquivos de "ajuda" (helpers/keywords). Veja as seções abaixo para a estrutura de cada framework.
  6. Execute os Testes: Abra o terminal (ou prompt de comando) na pasta principal do projeto. Use os comandos básicos para iniciar a execução dos testes (esses comandos estão no README.md).
  7. Veja os Relatórios: Após a execução, o framework irá gerar relatórios que mostram quais testes passaram e quais falharam. O README.md também indicará onde encontrar esses relatórios.

🏆 Boas Práticas

Para que seus testes de automação sejam fáceis de entender, rápidos de rodar e simples de manter, algumas boas práticas são muito importantes:

✅ Reutilização é Fundamental!

Imagine que em vários testes você precisa fazer login no sistema ou preencher os mesmos campos de um formulário. Repetir os mesmos passos em todos os testes significa que, se algo mudar (por exemplo, o campo de login mudar de nome), você terá que corrigir essa mudança em VÁRIOS lugares. Isso dá muito trabalho e aumenta a chance de erros.

A solução é criar um "comando" (Cypress/Playwright) ou uma "Keyword" (Robot Framework) para essas ações reutilizáveis. Funciona como criar um "atalho":

  • Você define os passos do login (digitar usuário, digitar senha, clicar em entrar) UMA VEZ em um arquivo de "ajuda".
  • Nos seus testes, em vez de repetir todos os passos, você apenas "chama" esse comando/keyword de login.

Benefício Principal: Se a tela de login mudar, você só precisa corrigir a definição do comando/keyword de login em UM lugar! Todos os testes que usam esse comando/keyword serão atualizados automaticamente. Isso economiza muito tempo e esforço na manutenção dos testes.

✨ Outras Dicas Importantes:

  • Use Seletores Estáveis: Ao identificar elementos na tela (botões, campos), prefira usar IDs (#meu-id), atributos data-testid ([data-testid="botao-salvar"]) ou nomes ([name="username"]) em vez de caminhos baseados na posição (xpath complexos ou CSS longos). Eles mudam menos se o layout da página for ajustado.
  • Testes Focados: Cada teste deve verificar uma coisa específica. Evite testes muito longos que misturam muitas funcionalidades diferentes. Testes menores são mais fáceis de entender quando falham.
  • Nomes Claros: Dê nomes descritivos aos seus testes e às suas keywords/comandos (ex: "Validar Mensagem de Erro ao Logar com Dados Inválidos", "Preencher Formulário de Cadastro"). Isso ajuda a entender o que cada parte faz sem precisar ler o código detalhado.
  • Leia a Documentação Gerada: Os arquivos README.md e FLUXO_DE_USO.md no template contêm informações valiosas específicas do projeto gerado. Leia-os!

Entendendo a Estrutura (Para Não Técnicos)

Cada template tem seus arquivos organizados de forma diferente, mas a ideia é sempre separar os testes em si das funcionalidades de "ajuda" que eles usam. Não se preocupe em entender todos os detalhes do código, foque em saber onde encontrar cada coisa:

Cypress 🌲

Estrutura de arquivos:

📁 cypress/ - Tudo relacionado ao Cypress.
📄 e2e/ - **Seus testes ficam aqui.** Cada arquivo .cy.ts é um conjunto de testes.
📄 support/ - Contém os arquivos de "ajuda".
📄 helpers.ts - Funcionalidades de ajuda menores (ex: "preencher campo").
📄 commands.ts - Combina helpers para criar "comandos" maiores e fáceis de usar (ex: "fazer login").
📄 cypress.config.ts - Configurações gerais do Cypress. **Aqui você muda a URL base!**

Em resumo: Crie ou edite seus testes em cypress/e2e/. As ações complexas são definidas em cypress/support/. Configure a URL em cypress.config.ts.

Playwright 🎭

Estrutura de arquivos:

📁 tests/ - **Seus testes ficam aqui.** Cada arquivo .spec.ts é um conjunto de testes.
📁 helpers/ - Onde ficam os arquivos de "ajuda".
📄 page-helpers.ts - Contém funcionalidades de ajuda para interagir com a página de forma organizada.
📄 playwright.config.ts - Configurações gerais do Playwright. **Aqui você muda a URL base!**

Em resumo: Crie ou edite seus testes em tests/. As ações de interação com a página são definidas em helpers/. Configure a URL e navegadores em playwright.config.ts.

Robot Framework 🤖

Estrutura de arquivos:

📁 tests/ - **Seus testes ficam aqui.** Cada arquivo .robot é um conjunto de testes.
📁 resources/ - Onde ficam os arquivos de "ajuda" chamados Keywords.
📄 helpers.robot - Keywords básicas (ex: "Preencher Campo", "Clicar No Elemento").
📄 common.robot - Keywords que combinam helpers para ações comuns (ex: "Fazer Login").
📄 variables.robot - Onde você define informações que mudam pouco. **Aqui você muda a URL base!**

Em resumo: Crie ou edite seus testes em tests/. As ações reutilizáveis são definidas como Keywords em resources/. Configure a URL e outras variáveis em variables.robot.

Próximos Passos

  • Leia o arquivo README.md na pasta do projeto baixado para instruções mais detalhadas sobre instalação e execução.
  • Se quiser entender mais a fundo como usar os helpers/commands/keywords, veja o arquivo FLUXO_DE_USO.md.
  • Comece a escrever seus próprios testes nos arquivos .cy.ts, .spec.ts ou .robot na pasta de testes! Use os exemplos existentes como base.

Esperamos que estes templates ajudem você a automatizar seus testes de forma mais eficiente!
